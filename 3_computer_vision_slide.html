功能 01

影像分類
(Image Classification)

「這張圖片的主題是什麼？」

核心概念

將影像歸類到一個或多個標籤 (Labels/Tags)。

多類別 (Multi-class) 互斥分類：這是一隻貓 (不是狗)。
多標籤 (Multi-label) 多個屬性：這是一隻貓，且是黑色的。
標籤：Cat (99%)
Car
Bike
回傳座標 (Bounding Box)
與類別
功能 02

物件偵測
(Object Detection)

「東西在哪裡？有多少個？」

  • 識別位置： 回傳物件的邊界框座標 (X, Y, Width, Height)。
  • 計數： 計算圖片中有幾輛車、幾個人。
  • 考題提示： 看到「Bounding Box (邊界框)」、「位置」、「座標」關鍵字，必選物件偵測。
功能 03

語意分割
(Semantic Segmentation)

「以像素為單位的精確分類。」

核心概念

不再是畫框框，而是將圖片中的每一個像素 (Pixel) 都歸類。

常見應用：自動駕駛

像素 1-1000 = 道路 (Road)
像素 1001-1200 = 車輛 (Car)
像素 1201-1500 = 天空 (Sky)

功能 04

光學字元辨識
(OCR)

「讀取圖片中的文字。」

Read API

Azure 最新的 OCR 技術，能夠讀取：

  • 印刷體
  • 手寫體
  • 多種語言與混合語言

應用場景

數位化紙本文件、讀取路牌、車牌辨識。

功能 05

臉部偵測與辨識
(Face Analysis)

「偵測人臉、分析屬性、識別身分。」

臉部偵測

回傳人臉的座標框。

屬性分析

分析年齡、情緒 (開心/生氣)、是否有戴眼鏡、口罩、化妝等。

臉部識別/驗證

這張臉是「張三」嗎？(驗證)；這張臉是誰？(識別)。

Azure 電腦視覺服務

如何選擇正確的服務？

1. Azure AI 視覺 通用型：預先訓練好的模型，直接用。
2. 自訂視覺 客製化：用「自己的照片」訓練模型。
3. 臉部服務 專業人臉：更進階的人臉辨識與驗證。
4. 文件智慧 文件專用：從表格、發票中提取結構化資料。
服務 01

Azure AI Vision

(舊稱 Computer Vision)

特點：預先訓練 (Pre-trained)

你不需要自己訓練模型，微軟已經訓練好了。

主要功能：

  • 分析影像： 產生描述、標籤、偵測物件。
  • OCR (Read)： 讀取圖片中的文字。
  • 縮圖生成： 智慧裁切 產生縮圖。
考題： 「為圖片產生自動描述」、「讀取路牌文字」 → 選 Azure AI Vision。
服務 02

Custom Vision
(自訂視覺)

「訓練你自己的模型」

特點：自帶數據 (Bring Your Own Data)

當 Azure AI Vision 無法識別你的特殊物件時使用。

1. 上傳圖片

例如：公司特定的產品零件。

2. 標記 (Label)

手動框出零件位置或分類。

3. 訓練 (Train)

雲端自動訓練。

4. 預測 (Predict)

發佈 API 進行預測。

考題： 「識別特定的花種」、「分辨自家產品是否有瑕疵」 → 選 Custom Vision。
服務 03

Face API

專門處理人臉的高階服務

與 Azure AI Vision 的區別？

Azure AI Vision 可以做基本的「臉部偵測」(哪裡有臉)，但 Face API 提供更深入的功能：

  • 臉部驗證 (Verification)： 1對1 比對，確認是不是同一個人 (如手機解鎖)。
  • 臉部識別 (Identification)： 1對多 搜尋，從資料庫中找出這個人是誰。
  • 相似臉部搜尋 (Find Similar)： 找出長得像的人。
  • 分組 (Grouping)： 將相似的臉分在同一組。
服務 04

Form Recognizer
(Document Intelligence)

文件智慧服務：從文件中提取結構化資料。

關鍵字：鍵值對 (Key-Value Pairs)

它不僅僅是讀取文字 (OCR)，還能理解文件的「結構」。

能提取什麼？

  • 表格 (Tables)
  • 選取方塊 (Selection marks)
  • 鍵值對 (Key: Value)

常見預建模型

  • 發票 (Invoices)
  • 收據 (Receipts)
  • 名片 (Business Cards)
  • 身分證件 (ID Cards)
注意： 題目若提到「自動化處理收據」、「提取發票金額」 → 必選 Form Recognizer。
